# Resumen de Trabajo - 25 de Abril 2026

## Descripción del Proyecto

Aplicación de pedidos a domicilio (delivery) con Spring Boot 4.0.5, Java 21, Thymeleaf y MySQL.

---

## Funcionalidades Implementadas

### 1. Módulo de Pedidos
- Estados: PENDIENTE → POR_CONFIRMAR → CONFIRMADO → EN_PREPARACION → EN_CAMINO → ENTREGADO → CANCELADO (el cliente no ve cancelados)
- El cliente puede cancelar solo si está en estado POR_CONFIRMAR

### 2. Métodos de Pago
- Admin puede configurar cuentas bancarias y códigos QR en `/admin/metodo-pago`
- Cliente ve opciones: EFECTIVO, TRANSFERENCIA, DATAFONO en checkout

### 3. Notificaciones WhatsApp (wa.me)
- Botón UNICO "✓ Confirmar y Notificar" para admin
- Después de confirmar:
  - Botón "Enviar al Restaurante" (abre WhatsApp Web)
  - Enlace copiable para el cliente

### 4. UI/UX
- Menú hamburguesa para móvil
- "Mis Pedidos" solo visible para CLIENTE (no para ADMIN)
- Diseño mobile-first

## Estados del Pedido

| Estado | Cuándo cambia |
|--------|--------------|
| POR_CONFIRMAR | Cliente selecciona método de pago |
| CONFIRMADO | Admin confirma el pedido |
| EN_PREPARACION | Admin envía al restaurante |
| EN_CAMINO | Domiciliario lleva el pedido |
| ENTREGADO | Cliente confirma recepción |
| CANCELADO | Cliente cancela (solo si POR_CONFIRMAR) |

## Endpoints Principales

### Público
- `/` - Home
- `/productos` - Lista productos
- `/login`, `/registro` - Autenticación

### Cliente (CLIENTE)
- `/carrito` - Ver carrito
- `/carrito/checkout` - Confirmar pedido
- `/pedidos/mis-pedidos` - Mis pedidos (no cancelados)

### Admin (ADMIN)
- `/admin/productos` - CRUD productos
- `/admin/metodo-pago` - Métodos de pago
- `/admin/pedidos` - Lista pedidos
- `/admin/pedidos/{id}` - Detalle y acciones
